Section 1: 实现注册功能

首先，根据项目要求，为爬虫数据展示网站实现登录注册界面，没有注册的用户无法访问到网站中的内容

由于在中期的博客里面已经详细讲解了如路由方法，数据库操作，性能优化等，所以在这次的博文里主要讲解功能的实现，而非一些技术上的细节

在实现之前，有一些概念和编程模式需要了解，需要了解的是React表单的使用，以下为一个示例

var Input = React.createClass({
  getInitialState: function() {
    return {value: 'Hello!'};
  },
  handleChange: function(event) {
    this.setState({value: event.target.value});
  },
  render: function () {
    var value = this.state.value;
    return (
      <div>
        <input type="text" value={value} onChange={this.handleChange} />
        <p>{value}</p>
      </div>
    );
  }
});

ReactDOM.render(<Input/>, document.body);

上面代码中，文本输入框的值要定义一个 onChange 事件的回调函数，通过 event.target.value 读取用户输入的值。

接下来就是用户登录/注册界面的实现案例，用户需要注册一个账号才能继续在数据查询网站的访问，登陆/注册界面如下所示

实现方式为利用React组件功能与属性，如下控制注册流程，使用validateFields可以帮助我们实现表单功能

this.props.form.validateFields((err, values) => {
  if (!err) {
    Api.post('/users/register', values)
      .then((res) => {
      if (res.data) {
        message.success('wegsite register success!');
        history.push('/login'); 
      }
    })
      .catch((err) => {
      console.log(err);
      message.error(err.message);// print err
    });
  }
});

在 Form.Item 中设置校验规则，在表单提交或表单输入变化时，通过执行 this.props.validateFields来实现表单值的校验。 在 Form.Item 中放置一个被 getFieldDecorator 注册的表单控件（子元素）来实现表单控件与表单的双向绑定，实现对表单值的收集。

紧接着使用html表单

Section 2: 实现登陆功能

注册后即可登陆，必须填写正确的账号密码才可以登录

此处需要使用JWT，JSON Web Token（缩写 JWT）是目前非常流行的跨域认证解决方案，JWT的原理是，服务器认证以后，生成一个 JSON对象，发回给用户。随后，用户与服务端通信的时候，都要发回这个 JSON 对象。服务器完全只靠这个对象认定用户身份。为了防止用户篡改数据，服务器在生成这个对象的时候，会加上signature（基于密钥）

const jsonwebtoken = require('jsonwebtoken');

JWT的介绍可以看阮一峰的博客 JSON Web Token 入门教程，里面有更加详尽的解释

async login(req, res) {
    const user = await User.findOne(req.body);
    if (!user) {
      res.status(404).json({ msg: 'error! Username or Password Invalid' });
    }
    const { _id, name } = user;
    const token = jsonwebtoken.sign({ _id, name }, scrects, { expiresIn: '1d' });
    res.status(201).json({ token, name });
  }


export const userLogin = (params) => {

Section 4: 信息加密处理

如之前的数据库内信息所见，我们的用户名和密码都是明文传播的，这在web应用中是极度不安全的，如下所示

为此，我们必须使用加密方法来保障安全。

在一切开始之前，我们需要借助crypto-js这个npm包

npm install crypto-js

安装好后，我们先尝试一下常用的md5加密方法

const cry = require('crypto-js') // required
const password = '123456';

const pwd = cry.MD5(password).toString();
//console.log(pwd);

此时可以得到MD5加密后的密码：e10adc3949ba59abbe56e057f20f883e，我们需要将之前的明文存储改为这类密文存储。

Remark：还支持其他形式的加密，都可以使用crypto-js来实现，简单做一下介绍，如下所示

首先是sha1加密

const cry = require('crypto-js')
const password = '123456';
const pwd_sha1 = cry.SHA1(password).toString();

此时可以得到sha1加密后的密码：7c4a8d09ca3762af61e59520943dc26494f8941b

也可以选择AES加密，AES指代高级加密标准（Advanced Encryption Standard），是美国联邦政府采用的一种区块加密标准。

// AES加密 
// arg1为需要加密的内容
// arg2为秘钥
const secretKey = 'def123';

const pwd_aes = cry.AES.encrypt(password, secretKey).toString();

此时可以得到AES加密后的密码：U2FsdGVkX19vthLwq1V+zSr7zifbqTBPoBQNCe92hx0=

总之，不论使用哪种加密方式，都比直接明文传输要好很多（我记得20年出过好几个大型互联网平台密码泄漏的事故…），然后看下加密后的效果吧

可见，想破解这些密码是相对困难的

至此，我们成功完成了用户登陆、注册和加密功能的开发。
