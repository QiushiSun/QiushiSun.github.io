Section 1: 实现日志记录

实现日志功能，我们需要使用npm module morgan

morgan是express默认的日志中间件，也可以脱离express，作为node.js的日志组件单独使用。

const logger = require("morgan");

借助middleware保存信息，实现如下

server.use(logger(function (tokens, req, res) {
  var request_time = new Date();
  var request_method = tokens.method(req, res);
  var request_url = tokens.url(req, res);
  var status = tokens.status(req, res);
  var remote_addr = tokens['remote-addr'](req, res);
  // 通过sql写入mysql
  logDAO.userlog([request_time,request_method,request_url,status,remote_addr], function (success) {
    console.log('logging successful！');
  })
}))

值得注意的是，如果面对大量用户的情形，日志文件可能会被拖的非常长，可以借助file-stream-rotator来进行日志划分，但因为我们是存关系型数据库所以不需要过多考虑这方面的问题，若存本地文件的话可以参考 日志切割

我在网上查阅了更加进阶的实现方式，即基于sentry（哨兵）的错误日志处理，虽然在期末周没有更多的精力去实现，但依然做一个比较简单的了解。Sentry可以做更多与异常/日志处理的工作，比如敏感信息过滤，版本跟踪，关键字查找与统计，受bug影响用户统计，权限管理等。而且可以提高查找效率。

更具体的内容，可以参考下面这篇文章 Sentry - 处理异常日志的正确姿势，从我个人角度而言可能会在以后的大型项目web开发中尝试这种方法实现日志

Section 3: 地理位置信息

在课程结束时老师提及了JavaScript获取记录用户访问时的地理位置信息的方式，在此做一点简单的尝试.

获取地理信息位置主要是使用Geolocation API，Geolocation 接口是一个用来获取设备地理位置的可编程的对象，它可以让Web内容访问到设备的地理位置，这将允许Web应用基于用户的地理位置提供定制的信息。说实话：其实Geolocation 就是用来获取到当前设备的经纬度（位置）

从编程角度而言，主要使用

Geolocation.getCurrentPosition()这个函数来确定设备的位置并返回一个携带位置信息的 Position 对象。

代码如下

function getLocation(){
if (navigator.geolocation){
  navigator.geolocation.getCurrentPosition(showPosition);
}else{
  x.innerHTML="Error!Unsupported browser/blocked access";} //可以先打印在屏幕上试试
}

通过返回的Position对象，可以知道当前经纬度

function showPosition(position)
{
	latitude_coords = Math.floor((position.coords.latitude*100)/100);
	longitude_coords = Math.floor((position.coords.laongitude*100)/100);
	x.innerHTML="latitude: " + latitude_coords + 
	"<br>longitude: " + longitude_coords;	
}

随后获得一个返回的Position对象，显然是上海市的经纬度坐标（我在本机试了一下）
