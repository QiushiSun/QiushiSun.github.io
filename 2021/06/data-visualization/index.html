Section 1: Echarts

ECharts，缩写来自Enterprise Charts，商业级数据图表，一个纯Javascript的图表库，可以流畅的运行在PC和移动设备上，兼容当前绝大部分浏览器，提供直观，生动，可交互，可高度个性化定制的数据可视化图表。创新的拖拽重计算、数据视图、值域漫游等特性大大增强了用户体验，赋予了用户对数据进行挖掘、整合的能力。

如上图所示在中期开发中已经使用过Echarts进行词云图展示，现在使用其他类型的图表做更加量化的展示。

因为提取/查询相关数据的sql、mongodb查询以及静态数据（Json格式）存储等过程和中期时无异，且代码比较冗杂，所以不在此赘述，若感兴趣可以回看中期项目的最后一篇文章，本次主要展现可视化的实现流程。

Section 2: 饼状动图的制作

Midterm时使用的Echarts为静态页面，在最终项目中尝试一下使用Echarts动图来展示数据。

饼状图适合直观展示各类数据的占比，从豆瓣影评中挑选出以下几类电影参与可视化：奇幻，动作，爱情，恐怖，剧情，数据依然是之前查询整理好的数据

实现动态效果的主要部件就是设置一个Interval来改变单个饼状图分块的强调时间，如下所示

setInterval(function () {
        var dataLength = option.series[0].data.length;
        // 这里需要把之前的高亮强调取消
        myChart.dispatchAction({
            type: 'downplay',
            seriesIndex: 0,
            dataIndex: app.CurIdx
        });
        app.CurIdx = (app.CurIdx + 1) % dataLength;
        // 强调当前的饼图模块，高亮当前的小块
        myChart.dispatchAction({
            type: 'highlight',
            seriesIndex: 0,
            dataIndex: app.CurIdx
        });
        // 显示tooltip，否则只会有高亮而非有文字的展示效果
        myChart.dispatchAction({
            type: 'showTip',
            seriesIndex: 0,
            dataIndex: app.CurIdx
        });
    }, 2000); // 此处的2000为ms，表示单个interval的长度

此处的颜色可以通过一个简单的js函数来实现，colorList列表中的颜色会被饼状图依次使用

color: function(params) {
                    var colorList = [
                    '#2980B9', '#1ABC9C', '#F1C40F','#F39C12 ','#A569BD'
                    ];
                    return colorList[params.dataIndex]
                }

Section 3: Echarts Heat-Map

在本版块，我们将尝试Echarts地图坐标系（参考了mooc的示范）

移动互联网时代，地理位置信息已经成为许多数据分析的重要参考依据，基于地图的图表需求也随之增长。ECharts 提供了一套功能完备的地图坐标体系，我们在此做一个简单尝试。

Heat-Map简介：某种程度上 ECharts 中的地图坐标系可以看做一种特殊直角坐标系，它将现实世界的地理平面映射为二维平面，通过经纬度系统实现坐标定位，通过线段按规则勾画出各个独立的地理、行政区域。

Remark：这里所使用的的地图信息数据是通过 json 格式定义，Echarts 曾提供国内地图信息，但由于不符合国家《测绘法》规定，目前和中国相关的地图信息都已经被枪毙/Echarts主动下架了。（哭了，我主要的数据都是国内新闻网站的）

Section 4: D3

D3简介：

前面都在使用Echarts工具开发数据可视化功能，现在尝试一下D3，因为一些图的功能是重复，所以把这个块单独做一下展示。

简介：D3.js 是一个可以基于数据来操作文档的 JavaScript 库。可以帮助你使用 HTML, CSS, SVG 以及 Canvas 来展示数据。D3 遵循现有的 Web 标准，可以不需要其他任何框架独立运行在现代浏览器中，它结合强大的可视化组件来驱动 DOM 操作。优势如下

  1. D3 基于svg图，因此对图像进行放大不会失真。

  2. D3 的步骤相对来说较多。对初学者不方便、也不好理解，但能够制作出更加精密的图形。

  3. 开发的图表形态多变

D3的官网链接: D3

SVG简介：

稍微再简介一下什么是SVG图，可能一直使用LaTeX的同学会对这个比较熟悉，做各种模版的图标和Title图标都会尝试使用SVG，而非一般的存储格式。SVG的全称为Scalable Vector Graphics，是一种使用XML技术描述二维图形的语言。SVG最大的特点是其为矢量图，随着网页的放大和缩小，图像不会失真。此外，这类图片可以被搜索、索引。最后，SVG 图像比 JPG 或 PNG 图像更灵活，而且我们可以使用 CSS 和 JavaScript 与它们进行交互。
