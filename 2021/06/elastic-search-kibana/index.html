Elastic Search + Kibana

在这篇文章中，我们对Elastic Search + Kibana数据展示做一个简单的了解

安装与部署

我们先要安装Elastic Search，下载地址为 Download Elasticsearch 再安装Kibana插件，地址为 Installing Kibana（这两个包都挺大的）

完成上述步骤以后，若Elastic Search使用默认配置的话，我们需要调整Kibana的配置文件，访问到启动的Elastic Search服务，参考下图的配置

分别cd到目录下，使用两个Terminal分别使用命令 ./bin/elasticsearch和./bin/kibana启动服务

在本地默认就是使用本地集群的方式启动，如上图所示

启动Elastic Search + Kibana

二者均启动成功后，我们即可通过访问 localhost:5061来访问Elastic Search + Kibana了，Home页面如下

我们下载一份Demo数据集后，可以看到Elastic Search + Kibana的功能非常丰富，涵盖了之前开发的所有可视化方式

导入数据

接下来是导入我们的爬虫数据，我们可以直接使用MySQL Workbench或者Navicat的Wizard来导出关系表，这部分在期中报告已经详细讲解，请参考期中阶段的博文 Web Crawler Part6

我们还需要在Dev Tools中建立数据表，获得所有属性，表中为我们的新闻数据

然后可以通过Data Visualizer部分导入数据，简单的数据预览如下所示，可以看到这些数据被采集的信息，如时间，还有数据来源等信息，下图为对我们的爬虫所采集的数据的预览

Remark:导入数据的操作通过本地的curl命令也可以实现

具体展示方式与上述demo一致，但因为数据没有样例的元数据那么多元，所以没有体现出诸多炫酷的功能

地图数据探索

我们在可视化那一章的地图信息可视化开发中遇到了瓶颈，但好在Elastic Search + Kibana能非常完善地解决地图相关的需求，因为我手头爬虫采集的数据在地理分布上主要是国内的，ES并没有提供这方面的地图支持，所以还是借助Demo简单了解一下

可见Es的地图上可视化的功能比Echarts原生的一些功能还是要精美不少的，受数据的限制，只能先借示例数据集做一个小小的展示了，期待日后的开发里更加深入地使用这些功能。

至此我们初步尝试了Elastic Search + Kibana进行数据展示，可见这种方式非常高效，只需要结构化的文本即可进行数据分析和展示的工作
