Web Crawler Part1: Some preparations

Web Crawler Part1

我们将用几篇博客来研究研究一个基于Node.js的爬虫是如何运作的

Section1: Prepare Your MySQL Database

mysql安装地址：https://dev.mysql.com/downloads/

进入上述网址安装一个MySQL数据库用于存储我们从互联网爬取的各类信息（当然，你也可以把它们存在文本文件中，不过在进行CRUD操作时就会略显麻烦，还是让数据库代劳吧。）

我们需要：

  1. MySQL Community Server（数据库服务器本身）
  2. MySQL Shell（用于在命令行操作数据库，MacOS版在安装MySQL Server后自行配置即可）
  3. 可选：MySQL Workbench （一个可视化界面，就像MongoDB的MongoDB Compass）

安装完成后可以配置 root 用户，并设置密码，然后使用如下命令登陆数据库：

mysql -u root -p

**Remark：**输入密码时，密码是不会显示在命令行的，只管输入即可。

执行上述命令吼，你可以在 mysql> 提示符执行SQL命令，退出请输入 “quit”

为了方便接下来的实验，进入mysql后我们创建一个数据库crawl,然后再创建一个表fetches用于存储信息，我们可以直接执行如下SQL语句：

CREATE TABLE `fetches` (
  `id_fetches` int(11)  NOT NULL AUTO_INCREMENT,
  `url` varchar(200) DEFAULT NULL,
  `source_name` varchar(200) DEFAULT NULL,
  `source_encoding` varchar(45) DEFAULT NULL,
  `title` varchar(200) DEFAULT NULL,
  `keywords` varchar(200) DEFAULT NULL,
  `author` varchar(200) DEFAULT NULL,
  `publish_date` date DEFAULT NULL,
  `crawltime` datetime DEFAULT NULL,
  `content` longtext,
  `description` longtext,
  `createtime` datetime DEFAULT CURRENT_TIMESTAMP,
  PRIMARY KEY (`id_fetches`),
  UNIQUE KEY `id_fetches_UNIQUE` (`id_fetches`),
  UNIQUE KEY `url_UNIQUE` (`url`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8;

id_fetches被设定为主键，之后我们会用到一些索引技术来加速我们的查询功能

Section2: Use Npm to Install MySQL Module

我们要在Node.js环境下使用MySQL数据库，安装相应的模组显然是必不可少的，在项目的工作路径下,使用命令行输入如下命令：

npm install mysql

即可直接安装mysql module

Section3: Trouble Shooting

我个人是在macOS系统下进行开发的，在配置MySQL数据库时会遇到一些问题，在此

Case1: mysql环境变量错误

在Mac端的terminal或者是iterm中输入mysql -u root -p，出现：zsh: command not found: mysql的提示，那么就代表当前MySQL的环境变量配置有误，可以通过以下方式解决：

  1. cd ~ 进入根目录
  2. sudo vim .bash_profile 打开bash配置文件
  3. 末尾追加：export PATH=${PATH}:/usr/local/mysql/bin
  4. source .bash_profile
  5. vi ~/.zshrc
  6. 末尾追加：export PATH=${PATH}:/usr/local/mysql/bin
  7. source ~/.zshrc

重新使用命令 mysql -u root -p 即可

Case2: Rethrow non-MySQL errors

在使用Node.js的MySQl模组时候，可能会有

Error: ER_NOT_SUPPORTED_AUTH_MODE: Client does not support authentication protocol requested by server; consider upgrading MySQL client flush privileges;

的报错，这是因为不同版本认证机制不同造成的，解决方案如下

  1. mysql -u root -p

  2. 执行语句 alter user 'root'@'localhost' identified with mysql_native_password by yourpwd; mysql> alter user ‘root’@‘localhost’ identified with mysql_native_password by ‘123456’; Query OK, 0 rows affected (0.43 sec)

  3. 再执行语句 flush privileges; mysql> flush privileges; Query OK, 0 rows affected (0.05 sec)

  4. mysql> quit Bye

上述两种错误是MacOS端配置MySQL数据库时的常见问题。
